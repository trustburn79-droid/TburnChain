3️⃣ 핵심 원칙 쉬운 설명
원칙: "토큰은 스마트 컨트랙트가 관리, 사람은 승인만"
┌─────────────────────────────────────────────────────────────────────────────┐
│                         무슨 뜻인가요?                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   ❌ 잘못된 방식 (위험)                                                     │
│   ─────────────────────                                                     │
│                                                                             │
│   "사장님 지갑에 100억 토큰 다 넣어두고, 필요할 때마다 꺼내 쓴다"          │
│                                                                             │
│   문제점:                                                                   │
│   • 사장님 비밀번호 해킹당하면? → 100억 전부 도난                          │
│   • 사장님이 마음 바뀌면? → 마음대로 빼돌릴 수 있음                        │
│   • 사장님이 사고 당하면? → 아무도 접근 못함                               │
│                                                                             │
│   ─────────────────────────────────────────────────────────────────────    │
│                                                                             │
│   ✅ TBURN 방식 (안전)                                                      │
│   ──────────────────────                                                    │
│                                                                             │
│   "컴퓨터 프로그램(스마트 컨트랙트)이 금고지기 역할"                        │
│   "사람은 '이거 맞아요'라고 승인만 함"                                      │
│                                                                             │
│   작동 방식:                                                                │
│   1. 프로그램: "오늘 투자자A에게 100만 토큰 줄 차례입니다"                 │
│   2. 사람들: "네, 맞습니다" (7명 이상이 승인)                              │
│   3. 프로그램: "3일 후에 보내드립니다" (대기 시간)                         │
│   4. 3일 후: 자동으로 투자자A에게 전송                                     │
│                                                                             │
│   장점:                                                                     │
│   • 프로그램은 감정이 없음 → 마음대로 못 바꿈                              │
│   • 정해진 규칙만 따름 → 예측 가능                                         │
│   • 여러 사람 승인 필요 → 혼자 못 빼돌림                                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

│                                                                             │

│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────┐      │
│   │                                                                 │      │

┌─────────────────────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────────────────────┤

│                                                                             │

│   ┌─────────────────────────────────────────────────────────────────┐      │
│   │                                                                 │      │
│   │   작은 금액 (100만 토큰 이하)                                   │      │
│   │   ───────────────────────────                                   │      │
│   │   • 승인 후 2일(48시간) 기다려야 출금                          │      │
│   │   • 2일 안에 "이상하다!" 발견하면 취소 가능                    │      │
│   │                                                                 │      │
│   │   큰 금액 (1억 토큰 이상)                                       │      │

│   │   • 승인 후 7일 기다려야 출금                                  │      │
│   │   • 7일 동안 모니터링 → 이상 시 취소                           │      │
│   │                                                                 │      │
│   │   초대형 금액 (5억 토큰 이상)                                   │      │
│   │   ───────────────────────────                                   │      │
│   │   • 승인 후 30일 기다려야 출금                                 │      │
│   │   • 한 달 동안 커뮤니티도 감시 가능                            │      │

│   └─────────────────────────────────────────────────────────────────┘      │
│                                                                             │
│   왜 안전한가?                                                              │
│   ────────────                                                              │
│   • 해커가 승인받아도 바로 못 가져감                                       │
│   • 대기 시간 동안 "해킹당했다!" 발견 가능                                 │
│   • 발견하면 즉시 취소하고 금고 잠금                                       │

5️⃣ 비상 상황 대응
5.1 긴급 정지 버튼
┌─────────────────────────────────────────────────────────────────────────────┐
│                         🚨 빨간 버튼 (긴급 정지)                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   비유: 지하철 비상정지 버튼                                                │
│   ───────────────────────────                                               │
│                                                                             │
│   "위험 상황 발생 시 누구나 누를 수 있음"                                   │
│   → 누르면 일단 모든 게 멈춤                                               │
│   → 상황 파악 후 재가동 결정                                               │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────┐      │
│   │                                                                 │      │
│   │   긴급 정지 발동 조건:                                          │      │
│   │                                                                 │      │
│   │   자동 발동 (컴퓨터가 알아서):                                  │      │
│   │   • 1시간에 1억 토큰 이상 빠져나가면 → 자동 정지               │      │
│   │   • 가격이 1시간에 50% 폭락하면 → 자동 정지                    │      │
│   │   • 해킹 의심 패턴 감지되면 → 자동 정지                        │      │
│   │                                                                 │      │
│   │   수동 발동 (사람이 누름):                                      │      │
│   │   • 11명 중 3명만 동의하면 즉시 정지 가능                      │      │
│   │   • "뭔가 이상하다!" 싶으면 일단 멈추고 보자                   │      │
│   │                                                                 │      │
│   │   재가동 조건:                                                  │      │
│   │   • 7명 이상 동의 + 24시간 대기                                │      │
│   │   • 무슨 일인지 파악하고 해결한 후에만 재가동                  │      │
│   │                                                                 │      │
│   └─────────────────────────────────────────────────────────────────┘      │
│                                                                             │
│   왜 중요한가?                                                              │
│   ────────────                                                              │
│   • 해킹 진행 중이라도 → 버튼 누르면 즉시 중단                             │
│   • 정지는 쉽게 (3명), 재가동은 어렵게 (7명+24시간)                        │
│   • "일단 멈추고 생각하자" 전략                                            │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

6️⃣ 열쇠 보관 방법
HSM (하드웨어 보안 모듈)
┌─────────────────────────────────────────────────────────────────────────────┐
│                         🔐 열쇠 보관 금고                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   비유: 은행 대여금고 vs 집 서랍                                           │
│   ────────────────────────────                                              │
│                                                                             │
│   ❌ 집 서랍에 열쇠 보관                                                    │
│   • 도둑이 집에 들어오면 열쇠 가져감                                       │
│   • 컴퓨터 해킹당하면 비밀번호 털림                                        │
│                                                                             │
│   ✅ 은행 대여금고에 열쇠 보관 (HSM)                                       │
│   • 특수 금고 안에 열쇠 보관                                               │
│   • 금고 자체가 방탄, 방화, 해킹 방지                                      │
│   • 열쇠를 꺼내지 않고 금고 안에서 서명                                    │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────┐      │
│   │                                                                 │      │
│   │   TBURN의 열쇠 보관 방식:                                       │      │
│   │                                                                 │      │
│   │   열쇠 1개를 11조각으로 나눔                                    │      │
